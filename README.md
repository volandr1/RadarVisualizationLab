# Лабораторна робота: Розробка додатку для візуалізації вимірювань радару

## Короткий опис роботи
**Мета роботи:** Розробити вебдодаток, який зчитує дані з емульованої вимірювальної частини радара, наданої у вигляді Docker-образу, та відображає задетектовані цілі на графіку в полярних координатах.

**Призначення основних компонентів:**
* **`index.html`**: Визначає структуру інтерфейсу користувача, містить контейнер для графіка та панель налаштувань.
* **`style.css`**: Забезпечує візуальне оформлення додатку у темній темі для імітації реального радарного дисплея.
* **`script.js`**: Містить основну логіку: встановлення WebSocket-з'єднання, розрахунок відстані за формулою, візуалізацію точок через Plotly.js та відправку API-запитів для зміни конфігурації.

---

## Інструкції для запуску проєкту

### 1. Вимоги до середовища
* **Docker**: Необхідний для запуску сервісу емуляції радара.
* **Браузер**: Будь-який сучасний веббраузер із підтримкою ES6+.
* **Інтернет**: Потрібен для завантаження бібліотеки Plotly.js через CDN.

### 2. Запуск емулятора (Backend)
Виконайте ці команди в терміналі для розгортання радарного сервісу:
```bash
# Завантаження образу з Docker Hub
docker pull iperekrestov/university:radar-emulation-service

# Запуск контейнера на порту 4000
docker run --name radar-emulator -p 4000:4000 iperekrestov/university:radar-emulation-service
 ```

## 3. Результаты работы

### Визуализация интерфейса
![Скриншот работы приложения](путь/к/твоему/скриншоту.png)

* **Визуализация целей**: Реализовано отображение объектов в полярных координатах, где положение каждой точки определяется углом зондирования (`scanAngle`) и рассчитанным расстоянием.
* **Математическая обработка**: Для перевода времени задержки сигнала в расстояние (км) применена формула:
    $$R = \frac{c \cdot t}{2}$$
    где $c$ — скорость света ($300,000$ км/с), а $t$ — время распространения сигнала.
* **Динамическое управление**: Добавлен интерфейс для изменения скорости вращения (`rotationSpeed`), количества измерений на оборот (`measurementsPerRotation`) и скорости целей (`targetSpeed`) через API-запросы к сервису.
* **Стилизация**: Внедрена цветовая индикация точек на графике в зависимости от мощности отраженного сигнала (`power`), что позволяет визуально различать интенсивность эхо-сигнала.

## Общий вывод

В ходе выполнения лабораторной работы были закреплены практические навыки работы с протоколом WebSocket для обработки потоковых данных в реальном времени и библиотекой Plotly.js для визуализации данных в полярной системе координат. Ключевым результатом стало создание производительного веб-интерфейса, способного обрабатывать и отображать динамические изменения параметров радара без задержек.

Анализ производительности показал, что выбранный стек технологий (HTML/JS) эффективно справляется с визуализацией при высокой частоте обновления данных, обеспечивая точность отображения целей в соответствии с теоретическими формулами радиолокации.
