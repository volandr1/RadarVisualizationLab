# Лабораторна робота: Розробка додатку для візуалізації вимірювань радару

## Короткий опис роботи
**Мета роботи:** Розробити вебдодаток, який зчитує дані з емульованої вимірювальної частини радара, наданої у вигляді Docker-образу, та відображає задетектовані цілі на графіку в полярних координатах.

**Призначення основних компонентів:**
* **`index.html`**: Визначає структуру інтерфейсу користувача, містить контейнер для графіка та панель налаштувань.
* **`style.css`**: Забезпечує візуальне оформлення додатку у темній темі для імітації реального радарного дисплея.
* **`script.js`**: Містить основну логіку: встановлення WebSocket-з'єднання, розрахунок відстані за формулою, візуалізацію точок через Plotly.js та відправку API-запитів для зміни конфігурації.

---

## Інструкції для запуску проєкту

### 1. Вимоги до середовища
* **Docker**: Необхідний для запуску сервісу емуляції радара.
* **Браузер**: Будь-який сучасний веббраузер із підтримкою ES6+.
* **Інтернет**: Потрібен для завантаження бібліотеки Plotly.js через CDN.

### 2. Запуск емулятора (Backend)
Виконайте ці команди в терміналі для розгортання радарного сервісу:
```bash
# Завантаження образу з Docker Hub
docker pull iperekrestov/university:radar-emulation-service

# Запуск контейнера на порту 4000
docker run --name radar-emulator -p 4000:4000 iperekrestov/university:radar-emulation-service
 ```

### 3. Запуск вебдодатка (Frontend)
1.  **Переконайтеся**, що файли `index.html`, `style.css` та `script.js` знаходяться в одній папці.
2.  **Відкрийте файл** `index.html` безпосередньо через будь-який сучасний веббраузер.
3.  **Додаток автоматично** встановить з’єднання з емулятором за адресою `ws://localhost:4000`.

## Результати роботи

### Візуалізація інтерфейсу
![Скріншот роботи додатка](https://github.com/volandr1/RadarVisualizationLab/blob/main/%D0%A1%D0%BD%D0%B8%D0%BC%D0%BE%D0%BA%20%D1%8D%D0%BA%D1%80%D0%B0%D0%BD%D0%B0%202026-04-18%20144211.png)

* **Візуалізація цілей**: Реалізовано відображення об’єктів у полярних координатах, де положення кожної точки визначається кутом зондування (`scanAngle`) та розрахованою відстанню.
* **Математична обробка**: Для переведення часу затримки сигналу у відстань (км) застосовано формулу:
    $$R = \frac{c \cdot t}{2}$$
    де $c$ — швидкість світла ($300,000$ км/с), а $t$ — час поширення сигналу.
* **Динамічне керування**: Додано інтерфейс для зміни швидкості обертання (`rotationSpeed`), кількості вимірювань на оберт (`measurementsPerRotation`) та швидкості цілей (`targetSpeed`) через API-запити до сервісу.
* **Стилізація**: Впроваджено колірну індикацію точок на графіку залежно від потужності відбитого сигналу (`power`), що дозволяє візуально розрізняти інтенсивність ехо-сигналу.

## Висновок

Під час виконання лабораторної роботи були закріплені практичні навички роботи з протоколом WebSocket для обробки потокових даних у реальному часі та бібліотекою Plotly.js для візуалізації даних у полярній системі координат. Ключовим результатом стало створення продуктивного веб-інтерфейсу, здатної обробляти та відображати динамічні зміни параметрів радара без затримок.

Аналіз продуктивності показав, що обраний стек технологій (HTML/JS) ефективно справляється з візуалізацією при високій частоті оновлення даних, забезпечуючи точність відображення цілей відповідно до теоретичних формул радіолокації.
